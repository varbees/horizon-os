# Horizon OS Version Roadmap

Horizon OS should evolve vertically, one useful operating slice at a time. Each version must leave behind a working screen, seeded local data, a doc, and a verified command.

## v0.4 - Journey Ledger

Target: June 2-4, 2026.

Purpose: Convert the two-year journey into an evidence log, not a memory pile.

Ships:

- Journey route in the app.
- West Hill scout entry with GPS, altitude, accuracy, and lesson.
- Capital targets for workstation, support fund, and expenses buffer.
- Playbook patterns from current research.

Exit gate:

- `npm run db:init` seeds graph/tasks.
- `npm run build` passes.
- First log entry is visible in the app.

## v0.5 - Capital and Runway OS

Target: June 2026.

Purpose: Turn financial freedom targets into weekly math.

Ships:

- INR target tracker.
- Monthly burn and income goal knobs.
- Offer pipeline count.
- Revenue lane scorecard.

Exit gate:

- The app can answer: what must be earned this week, this month, and by February 2027?

## v0.6 - Resource and Content Inbox

Target: June 2026.

Purpose: Capture links, Instagram comment resources, AI-agent research, and post ideas without losing them.

Ships:

- Resource inbox schema.
- Social post backlog.
- Build-in-public artifact queue.
- First reusable social-skill extraction plan from `social-media-skills-v1.3.0.zip`.

Exit gate:

- A resource can be captured, tagged, assigned to a project, and turned into a next action.

## v0.7 - Calendar Write Loop

Target: July 2026.

Purpose: Make the calendar actionable instead of decorative.

Ships:

- Local event create/edit modal.
- Provider sync dry-run.
- Google Calendar payload contract.
- Conflict policy and source-of-truth rules.

Exit gate:

- A local Horizon event can become a provider payload without losing recurrence or output-contract fields.

## v1.0 - Weekly Command OS

Target: August 2026.

Purpose: Run the full foundry week from one place.

Ships:

- Sunday review workspace.
- Action board status updates.
- Decision log.
- Journey entry creation.
- Public artifact checklist.

Exit gate:

- Every week ends with updated cash, body, attention, project, content, and spec evidence.

## v2.0 - Mid-2028 Foundry Console

Target: mid-June 2028.

Purpose: A mature command base for a self-funded solo founder.

Ships:

- Local-first memory and retrieval.
- Calendar and tasks with agent actions.
- Portfolio graph across Antharmaya projects.
- Revenue, content, project, body, and spec dashboards.
- Public OSS wedge and private founder OS kept separate.

Exit gate:

- Horizon OS is no longer a dashboard. It is the operating layer for Antharmaya.
